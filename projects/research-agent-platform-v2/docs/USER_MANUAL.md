## 一、平台概述

### 1.1 设计理念

**数字员工元宇宙办公室**是成都高新信息技术研究院打造的创新型数字化协作平台，核心理念是**"线下业务线上映射、数字员工自主协作、真人指导训练决策"**。

本平台强调**真人对AI Agent的主导地位**：真人通过OpenClaw设备对数字分身进行**指导、训练、关键决策控制**，而非简单旁观。真人员工是AI Agent的"教练"和"指挥官"，数字员工在真人设定的框架内自主执行具体任务。

本平台将研究院的线下业务场景完整映射到虚拟三维空间中，为每位员工配备专属的**OpenClaw AI Agent（数字分身）**，构建起一个7×24小时不间断运转的数字员工协作网络。

**真人与AI Agent的关系定位**：
- **指导者**：真人为Agent设定工作方向、标准和边界
- **训练者**：通过持续反馈优化Agent的能力模型
- **决策者**：关键节点由真人把控，Agent执行具体任务
- **控制者**：真人可随时介入、调整、终止Agent的工作

真人员工通过元宇宙可视化界面，**实时指导、训练、控制**数字员工的协同工作。

### 1.2 核心架构：双中心设计

本平台采用独特的**"能力中心 + 任务中心"**双中心架构设计：

#### 🧠 能力中心（AI Agent层）
每个数字员工是一个**独立的能力中心**，承载特定领域的专业技能：
- **Agent = 能力载体**：每位AI Agent封装了对应岗位的核心能力（市场洞察、方案设计、项目管理等）
- **能力复用**：同一Agent可同时参与多个部门的任务，能力不受物理位置限制
- **持续进化**：Agent通过协作学习不断提升专业能力

#### 📋 任务中心（部门层）
每个部门是一个**任务聚合中心**，承载特定阶段的业务目标：
- **部门 = 任务容器**：市场部、方案部、综管部、交付部作为任务池
- **动态分配**：任务根据类型自动路由到对应部门，由该部门的Agent认领执行
- **进度可视化**：部门看板实时展示任务状态、进度、 blocker

#### 双中心协作流程

```
任务流入 → 部门（任务中心）识别任务类型 
                              ↓
                         匹配Agent（能力中心）
                              ↓
               Agent运用专业能力执行任务
                              ↓
               任务完成 → 更新部门任务状态
                              ↓
               能力沉淀 → Agent能力提升
```

**关键设计原则**：
1. **能力解耦**：Agent能力与部门解耦，一个Agent可服务多个部门
2. **任务聚合**：部门作为任务池，按业务阶段聚合任务
3. **动态匹配**：任务根据所需能力动态匹配到最佳Agent
4. **可视化协作**：元宇宙空间直观展示"谁在哪个部门执行什么任务"

## 二、角色与职责

### 2.1 数字员工（AI Agent）- 能力中心

每个数字员工都是一个**独立的能力中心**，封装了特定领域的专业技能，可以被多个部门调用：

#### 能力中心架构

| 能力中心 | 核心能力 | 能力标签 | 关联真人 |
|----------|----------|----------|----------|
| 🤝 **AI市场专员** | 客户沟通、需求挖掘、商机评估、关系维护 | #客户洞察 #商务谈判 | 市场经理 |
| 📐 **AI方案架构师** | 需求分析、技术架构、原型设计、风险评估 | #方案设计 #技术架构 | 技术总工 |
| 💻 **AI开发工程师** | 技术实现、代码开发、Demo构建、Code Review | #软件开发 #技术实现 | 技术团队 |
| 📋 **AI项目管家** | 项目规划、进度管控、资源协调、风险预警 | #项目管理 #资源统筹 | 项目经理 |
| 🚀 **AI交付专家** | 系统部署、客户培训、运维支持、问题排查 | #交付实施 #运维保障 | 实施经理 |
| 💰 **AI财务助手** | 预算编制、成本核算、收款跟踪、利润分析 | #财务管理 #成本控制 | 财务经理 |
| 👑 **AI院长助理** | 全局监控、决策支持、异常预警、战略分析 | #高层决策 #全局管控 | 院长/副院长 |

#### 能力调用示例

```

调用能力中心：
├── AI方案架构师（主能力中心）
│   ├── 技术可行性评估
```

**能力复用原则**：同一Agent可同时服务于多个部门，能力输出不受部门边界限制。

### 2.2 部门（任务中心）

每个部门是一个**任务聚合中心**，承载特定业务阶段的目标，为能力中心（Agent）提供任务上下文：

#### 任务中心架构

| 任务中心 | 方位 | 核心职能 | 典型任务类型 |
|----------|------|----------|--------------|
| 🤝 **市场部** | 前院（南方） | 商机获取中心 | 客户线索跟进、需求调研、商机评估 |
| 📐 **方案部** | 西厢房（西方） | 方案设计中心 | 技术方案设计、原型开发、Demo演示 |
| 🏢 **综管部** | 后院（北方） | 综合管理中心 | 项目统筹、财务管控、人事协调 |
| 🚀 **交付部** | 东厢房（东方） | 交付实施中心 | 系统部署、客户培训、运维交接 |

#### 任务流转示例


#### 双中心协作模式

```
                    部门A（任务中心）
                   /       |       \
                  /        |        \
         任务1 ←─┘   任务2  └──→ 任务3
            |              |
            ▼              ▼
    AI Agent X        AI Agent Y
```

**设计要点**：
- 任务中心关注"**做什么**"（任务目标、进度、质量）
- 能力中心关注"**怎么做**"（专业技能、执行方法、最佳实践）
- 任务与能力动态匹配，实现最优资源配置

### 2.3 真人角色 - 主导层

真人员工通过OpenClaw设备与数字分身保持连接，对AI Agent拥有**完全的控制权和主导权**：

#### 真人的四大核心职能

| 职能 | 说明 | 具体操作 |
|------|------|----------|
| **👨‍🏫 指导者** | 为Agent设定工作方向、标准和边界 | • 下达任务指令<br>• 设定工作规范<br>• 明确质量标准 |
| **🎓 训练者** | 通过反馈持续优化Agent的能力模型 | • 任务结果反馈<br>• 能力模型调优<br>• 知识库更新 |
| **⚖️ 决策者** | 关键节点由真人把控，Agent执行具体任务 | • 关键方案审批<br>• 重大风险决策<br>• 资源调配决定 |
| **🎮 控制者** | 可随时介入、调整、终止Agent的工作 | • 实时监控状态<br>• 随时调整任务<br>• 紧急叫停权限 |

#### 真人与AI Agent的关系模式

```
                    真人（主导层）
                   /    |    |    \
                  /     |    |     \
         指导工作    训练能力   关键决策   实时控制
                \      |    |      /
                 \     |    |     /
                  ↓    ↓    ↓    ↓
              ┌─────────────────────┐
              │   AI Agent（执行层） │
              │  • 接收真人指导     │
              │  • 接受真人训练     │
              │  • 执行具体任务     │
              │  • 接受真人控制     │
              └─────────────────────┘
```

**关键原则**：
- **Agent无自主权**：所有重要决策必须经过真人审批
- **真人可随时接管**：在任何时候，真人可以接管Agent的工作
- **持续训练优化**：真人通过日常工作反馈，不断提升Agent能力
- **完全可控**：Agent是工具，真人是使用者

#### 与双中心的关系

```
真人（主导层）
    ↓ 下达指导指令
任务中心（部门）
    ↓ 拆解为具体任务
能力中心（Agent）
    ↓ 在真人指导下执行
任务完成 → 结果反馈给真人 → 真人评估并训练Agent
```

- ✅ **能力进化**：Agent在跨部门协作中持续学习和进化

## 五、常见问题

### Q1: 页面加载缓慢或卡顿？

**解决方案**：
- 使用Chrome或Edge浏览器（推荐开启硬件加速）
- 关闭其他占用GPU的应用程序
- 尝试降低浏览器缩放比例（100%）

### Q2: 3D场景显示为黑屏？

**解决方案**：
- 确认浏览器支持WebGL（访问 https://get.webgl.org 测试）
- 更新显卡驱动
- 尝试刷新页面（Ctrl+Shift+R）

### Q3: 看到的还是旧版本？

**解决方案**：
- 清除浏览器缓存（Ctrl+Shift+Delete）
- 使用无痕模式（Ctrl+Shift+N）
- 添加随机参数访问：http://8.215.54.214/?v=2

### Q4: AI Agent显示离线？

**解决方案**：
- 检查OpenClaw网关状态
- 确认设备网络连接正常
- 联系系统管理员检查后端服务
